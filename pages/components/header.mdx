# Header Component

The Header component provides the main navigation for the website with responsive mobile menu support.

## Location

`site/src/components/Navigation/Header.vue`

## Features

- Responsive design with mobile hamburger menu
- Active route highlighting
- Smooth transitions
- Accessible navigation with ARIA labels
- Emergency contact quick access

## Props

This component doesn't accept any props - it's a standalone navigation component.

## Usage

```vue
<template>
  <div id="app">
    <Header />
    <router-view />
    <Footer />
  </div>
</template>

<script setup>
import Header from '@/components/Navigation/Header.vue'
import Footer from '@/components/Navigation/Footer.vue'
</script>
```

## Responsive Behavior

### Desktop (1024px and above)
- Horizontal navigation bar
- All links visible
- Logo on the left
- Emergency link highlighted on the right

### Tablet (768px - 1023px)
- Condensed horizontal navigation
- Logo and key links visible
- Some links may collapse

### Mobile (below 768px)
- Hamburger menu icon
- Slide-out navigation drawer
- Full-screen menu overlay
- Touch-friendly spacing

## Accessibility

### ARIA Labels

```vue
<button
  class="mobile-menu-button"
  @click="toggleMobileMenu"
  aria-label="Toggle navigation menu"
  :aria-expanded="isMobileMenuOpen"
>
  <span class="hamburger-icon"></span>
</button>
```

### Keyboard Navigation

- All links are focusable
- Tab order follows visual flow
- Enter/Space activates links
- Escape closes mobile menu

### Screen Reader Support

```vue
<nav aria-label="Main navigation">
  <router-link
    v-for="link in navigationLinks"
    :key="link.name"
    :to="link.path"
    :aria-current="route.path === link.path ? 'page' : undefined"
  >
    {{ link.label }}
  </router-link>
</nav>
```

## Best Practices

1. **Sticky Position**: Header stays visible during scroll for easy navigation
2. **Active State**: Current route is visually highlighted
3. **Touch Targets**: Mobile links have minimum 44px height for accessibility
4. **Performance**: Menu state managed with simple ref, no complex state
5. **Close on Navigate**: Mobile menu automatically closes after navigation

## Customization

### Changing Navigation Links

Edit the `navigationLinks` array:

```javascript
const navigationLinks = [
  {
    name: 'Home',
    path: '/',
    label: 'Home'
  },
  // Add more links...
]
```

### Adjusting Mobile Breakpoint

Update the media query:

```css
@media (max-width: 767px) {
  /* Mobile styles */
}
```
