# Service Card Component

A reusable card component for displaying therapy services and offerings.

## Location

`site/src/components/Common/ServiceCard.vue`

## Features

- Clean card design with hover effects
- Icon support
- Title and description display
- Responsive layout
- Optional link/action support
- Accessible markup

## Props

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `title` | `String` | Yes | - | Service title |
| `description` | `String` | Yes | - | Service description |
| `icon` | `String` | No | `undefined` | Icon image URL or path |
| `link` | `String` | No | `undefined` | Optional link destination |

## Component Code

```vue
<script setup>
import { computed } from 'vue'

const props = defineProps({
  title: {
    type: String,
    required: true
  },
  description: {
    type: String,
    required: true
  },
  icon: {
    type: String,
    default: undefined
  },
  link: {
    type: String,
    default: undefined
  }
})

const emit = defineEmits(['click'])

const handleClick = () => {
  emit('click')
}

const isLink = computed(() => !!props.link)
</script>

<template>
  <component
    :is="isLink ? 'router-link' : 'article'"
    :to="link"
    class="service-card"
    @click="handleClick"
  >
    <div v-if="icon" class="service-icon">
      <img :src="icon" :alt="title" />
    </div>

    <h3 class="service-title">{{ title }}</h3>

    <p class="service-description">{{ description }}</p>
  </component>
</template>
```

## Responsive Design

### Desktop (1024px and above)
- Full padding and spacing
- Larger icon (60px)
- Standard font sizes

### Tablet (768px - 1023px)
- Slightly reduced padding
- Medium icon (50px)
- Maintained readability

### Mobile (below 768px)
- Compact padding
- Smaller icon (40px)
- Adjusted typography

```css
@media (max-width: 767px) {

  }
```

```css
}
```

## Best Practices

1. **Keep Content Concise**: Titles should be 2-4 words, descriptions 1-2 sentences
2. **Use Icons Consistently**: All cards in a group should have icons or none at all
3. **Accessible Hover States**: Ensure sufficient color contrast
4. **Loading States**: Consider skeleton loaders for dynamic content
5. **Equal Heights**: Use CSS Grid to maintain consistent card heights

## Testing

```javascript
        description: 'Test'

    expect(wrapper.emitted('click')).toBeTruthy()
  })
})
```
